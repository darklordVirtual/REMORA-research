## 2. Dependency Pinning

### Current state

- **`requirements-lock.txt`**: committed, 267 packages, generated by `pip freeze` on Python 3.14.0. File is UTF-16 LE encoded (Windows PowerShell artifact) — standard `pip install -r` handles this, but `diff` tools may show garbled output. Recommend regenerating on Linux with UTF-8.
- **`pyproject.toml`**: all runtime dependencies are declared as optional extras (`[llm]`, `[dev]`, etc.). The core `dependencies = []` entry means `pip install remora` installs nothing — all runtime deps are opt-in. This is intentional for a research library.
- **`uv.lock`**: not present. REM-015 explicitly notes this as out of scope for research-grade reproducibility.
- **SPDX/CycloneDX SBOM**: not present.
- **Docker image digests**: not pinned in `Dockerfile` or `docker-compose.yml`.

### Risk

A fresh environment built from `pip install -e ".[dev]"` without the lockfile will resolve to whatever is latest on PyPI at build time. The lockfile is advisory, not enforced by CI. This is the primary reproducibility gap.

### Recommendation

See Section 6 (Recommended Next Steps).

## 6. Recommended Next Steps

### P1 — High priority

1. **Re-encode `requirements-lock.txt` as UTF-8.** On Linux: `pip freeze > requirements-lock.txt`. This makes the file diff-readable and avoids encoding surprises in CI.

2. **Pin the lockfile in CI.** Add a step to `ci.yml` or `quality-gates.yml`:
   ```yaml
   - name: Install from lockfile
     run: pip install -r requirements-lock.txt && pip install -e ".[dev]" --no-deps
   ```
   This ensures CI uses the exact versions that were tested locally.

3. **Add commit-hash provenance to large result files** that currently lack it (e.g., `thermodynamic_eval_n500_calibrated_results.json`).

### P2 — Medium priority

4. **Generate `uv.lock`** if the project migrates to `uv` as package manager. Until then, the `pip freeze` lockfile is the best available option.

5. **Add artifact checksums to CI.** The `artifact_manifest_v1.md` in this directory provides a baseline. A CI step that verifies checksums of the immutable historical artifacts (those not regenerated on each run) would catch accidental mutations.

6. **Generate a minimal SBOM** using `pip-licenses` or `cyclonedx-bom` for the `[dev]` extra. This is low effort and satisfies external review requirements.

7. **Pin base Docker image by digest** in `deploy/docker-compose/Dockerfile` (e.g., `python:3.12-slim@sha256:...`).

### P3 — Low priority (research-grade, future work)

8. Full SPDX/CycloneDX SBOM with transitive dependency tree (out of scope per REM-015).
9. HSM/KMS for `REMORA_PDP_SIGNING_KEY` (out of scope per REM-022).
